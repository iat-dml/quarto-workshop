One Tool to Rule Them All

Quarto for documentation, presentations, manuscripts, and outreach — a unified workflow for academic communication.

Format Hands-on workshop
Audience Early-career & experienced researchers
Outputs covered HTML · PDF · DOCX · Slides · Dashboards

Start with the foundations → Jump to use cases

Quarto is an open-source scientific and technical publishing system created by Posit. It helps researchers move from scattered workflows to a more integrated way of writing, documenting, presenting, and sharing their work.

This workshop website is designed as a guided reference. You can follow it from top to bottom during a live session, or return to individual pages later when you need help with a specific output format. The core promise is simple: write once in plain text and render to html, pdf, docx, and revealjs slides — without rewriting any of the content.

Workshop idea

The core promise of Quarto is simple: write once in a plain-text source file and render to multiple professional outputs — HTML, PDF, DOCX, and slides — from one consistent authoring model.

1Who this workshop is for

This material is especially useful for:

  • early-career researchers building their first durable workflow
  • experienced researchers who want fewer disconnected tools
  • teams that care about reproducibility, transparency, and reuse
  • anyone who wants websites, manuscripts, presentations, and notes to feel part of the same system

3A taste of the workflow

Here is a tiny Quarto document. The same source becomes a styled HTML page, a print-ready PDF, or a Word file — depending on how you render it.

yaml + python 
---
title: "Soil moisture across plots"
author: "E. Researcher"
format:
  html: default
  pdf:   default
  docx:  default
---

## Quick look at the data

```{python}
import pandas as pd
df = pd.read_csv("plots.csv")
df.groupby("plot")["vwc"].mean().round(2)
```

Render the same file three ways from the terminal:

shell 
# a single source → three professional outputs
quarto render notes.qmd --to html
quarto render notes.qmd --to pdf
quarto render notes.qmd --to docx
Suggested path

If you are brand new, begin with What Is Quarto? and then continue to Markdown for Beginners. The first two sections take roughly 30 minutes combined.

5How to use this site

The workshop is organized in three parts:

  1. Start with the Foundations pages if Quarto is new to you.
  2. Move into the Academic Use Cases to see how one tool supports different research outputs.
  3. Finish with the Advanced Topics when you are ready to customise or scale your work.

Each use-case lesson has a separate guided exercise page. The lesson explains the workflow, and the exercise helps you practice it with an academic scenario.

Before you start

Make sure Quarto ≥ 1.5 is installed and that you can run quarto check in your terminal. If you hit issues, see the Installing Quarto page.

One source, many outputs

The same .qmd file you write today for a quick lab note can become a submission-ready PDF tomorrow — and a slide deck the week after. That is the workflow this workshop is built around.
